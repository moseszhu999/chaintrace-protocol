# Compliance Boundaries

ChainTrace is designed as an open proof and fact infrastructure project.

It should not be presented as a public investment opportunity, lending product, payment product, or token sale platform in the early stage.

## Core Boundary

ChainTrace helps users create verifiable proof records.

It does not, by itself, provide:

- investment returns
- lending services
- custody services
- payment services
- stablecoin issuance
- securities issuance
- public fundraising
- guaranteed financing

## Proof Infrastructure vs Financial Services

ChainTrace may help users prepare evidence such as:

- product proof
- shipment proof
- invoice proof
- buyer acceptance proof
- receivable proof pack
- business passport

But any actual financing, lending, factoring, stablecoin payment, or regulated financial service should be handled by properly licensed or compliant partners where required.

## Token Boundary

Any future token design should be evaluated carefully.

Safe early-stage framing:

- usage credits
- testnet token
- network utility
- anti-spam deposit
- agent service payment
- data verification incentive
- governance research

High-risk framing to avoid:

- guaranteed returns
- profit sharing
- dividends
- interest income
- token buyback promises
- financing proceeds for token holders
- public investment promises

## Privacy Boundary

ChainTrace is not a total transparency system.

Commercial privacy matters.

The protocol should support:

- file hashes instead of full public files
- metadata hashes instead of sensitive metadata
- selective disclosure
- off-chain storage
- encrypted files when needed
- permissioned views where appropriate

## Jurisdiction Boundary

ChainTrace should not target restricted jurisdictions for crypto-asset trading, token sales, stablecoin services, or financial services.

The project should clearly state that users are responsible for complying with local laws.

## Communication Principle

The safest public message is:

> ChainTrace is an open protocol for verifiable supply chain facts.

Avoid presenting the project as:

- a get-rich crypto project
- an ICO
- an investment product
- a shadow banking platform
- a way to avoid regulation

## Recommended Language

Use:

- verifiable proof
- selective disclosure
- trade evidence
- supply chain facts
- business passport
- AI-assisted verification
- infrastructure for honest small businesses

Avoid:

- guaranteed yield
- token moonshot
- anonymous finance
- no regulation
- bypass compliance
- profit distribution
