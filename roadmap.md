# ChainTrace Roadmap

This roadmap is intentionally staged from simple proof pages to infrastructure.

The long-term vision is a global supply chain fact layer. The starting point is a tool that lets a small business create a verifiable proof page in minutes.

## Phase 0 — Protocol Draft

Goal: define the conceptual and technical foundation.

- Publish manifesto
- Define core data model
- Define proof schema
- Define batch and event schema
- Define attestation model
- Define agent manifest concept
- Define compliance boundaries

## Phase 1 — Proof Page MVP

Goal: allow a user to create a verifiable proof page.

- Upload evidence
- Generate file hash
- Store file off-chain or externally
- Anchor hash on testnet
- Generate public proof page
- Generate QR code
- Verify proof by recalculating hash
- Add blockchain explorer link

## Phase 2 — Business Passport

Goal: help small businesses build portable, verifiable trade history.

- Create business profile
- Aggregate product proofs
- Aggregate shipment proofs
- Aggregate invoice proofs
- Display verified trade timeline
- Add buyer or partner attestations
- Add AI-generated summary
- Add shareable public link

## Phase 3 — AI Agent Layer

Goal: allow AI agents to cooperate around verifiable supply chain facts.

- Evidence Agent
- Trace Agent
- Finance Pack Agent
- Compliance Agent
- Integration Agent
- Agent manifest schema
- Third-party agent registration concept

## Phase 4 — API and SDK

Goal: allow existing systems and platforms to integrate ChainTrace.

- JavaScript SDK
- REST API
- Webhook bridge
- ERP integration examples
- SaaS platform integration examples
- Shopify-style plugin concept
- Developer documentation

## Phase 5 — Receivable Proof Network

Goal: prepare trade evidence for supply chain finance partners.

- Receivable proof pack
- Buyer confirmation flow
- Duplicate financing warning
- Risk summary
- Financier review interface
- Stablecoin or payment partner integration research

## Phase 6 — Open Ecosystem

Goal: enable independent applications to grow on top of the protocol.

- Agent marketplace concept
- Proof templates
- Industry-specific schemas
- Community schemas
- Governance research
- Long-term utility token research under legal review

## Near-term Priority

The next concrete milestone is:

> A working Proof Page MVP where a user can upload a file, generate a hash, anchor it on a testnet, and share a public verification page.
